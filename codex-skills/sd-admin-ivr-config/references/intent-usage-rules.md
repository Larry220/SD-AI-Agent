# Intent 使用规则总结

## 1. 核心定义

`intent` 表示当前这一次 AI 回复所命中的流程节点或挂机节点。

它不是客户画像，不是整通电话的最终评价，也不是客户历史意向的累计结果。

正确理解：

```text
当前客户回复 -> 命中当前 SOP 分支 -> 进入对应节点 -> 输出该节点绑定的 intent
```

错误理解：

```text
客户前面表现高意向 -> 后面所有回复都输出高意向成交类
```

## 2. Intent 的两种类型

### 2.1 非挂机节点 intent

非挂机节点用于继续推进对话。

这类节点的 `intent` 应输出当前节点 ID，例如：

| 场景 | intent |
| --- | --- |
| 开场破冰 | `stage_1_opening` |
| 活动权益说明 | `stage_2_value` |
| 安装版本确认 | `stage_3_version_confirm` |
| 微信与安装资料承接 | `stage_4_wechat_handoff` |
| 客户不方便接听异议处理 | `objection_busy` |
| 客户拒绝异议处理 | `objection_reject` |
| 收费疑问 | `objection_fee` |
| 公司资质疑问 | `objection_company` |
| 软件效果疑问 | `objection_effect` |
| 只要指标疑问 | `objection_indicator_only` |
| 没时间听课 | `objection_no_time_course` |

### 2.2 挂机节点 intent

挂机节点用于结束当前通话。

这类节点的 `intent` 只能输出以下四类之一：

| 挂机 intent | 使用条件 |
| --- | --- |
| `高意向成交类` | 当前实际触发高意向成交类挂机节点 |
| `待跟进留存类` | 当前实际触发待跟进留存类挂机节点 |
| `无意向终止类` | 当前实际触发无意向终止类挂机节点 |
| `异常场景应急类` | 当前实际触发异常场景应急类挂机节点 |

## 3. 最重要的判定规则

### 3.1 Intent 跟当前节点走

当前命中哪个节点，就输出哪个节点绑定的 `intent`。

不要根据客户之前说过的话去覆盖当前节点结果。

示例：

```text
客户前面愿意了解，后面说“我现在没空，晚点再说”
```

正确输出：

```text
嗯，那先帮您把免费名额保留到{变量:活动截止时间}，后面专属顾问会在微信上对接安装。您先忙，再见。{"intent":"待跟进留存类"}
```

原因：

最终触发的是“晚点再说”的待跟进挂机节点，不是高意向成交挂机节点。

### 3.2 前面状态不能锁死后面 intent

客户在一通电话中可能发生状态变化。

例如：

| 对话变化 | 最终命中节点 | 正确 intent |
| --- | --- | --- |
| 先感兴趣，后面说晚点再说 | 待跟进挂机节点 | `待跟进留存类` |
| 先拒绝，后面愿意了解 | 回到权益说明或安装确认节点 | 当前流程节点 ID |
| 先愿意装，后面最终拒绝 | 无意向挂机节点 | `无意向终止类` |
| 开场无应答，重复后仍无应答 | 异常挂机节点 | `异常场景应急类` |

### 3.3 不按意向强弱猜 intent

不要用“客户看起来比较有兴趣”或“客户好像不感兴趣”来猜 `intent`。

只允许按 SOP 分支和当前节点映射输出。

## 4. 四类挂机 intent 的使用规则

### 4.1 高意向成交类

只在当前通话已经完成转化闭环时输出。

典型触发条件：

1. 客户同意添加微信。
2. 客户愿意接收安装资料。
3. 客户无异议，接受后续专属顾问对接。
4. 客户反馈安装遇到问题，但已经接受微信一对一指导方案。

输出示例：

```text
嗯好，那我不耽误您时间了，后面专属顾问会在微信上跟您详细对接。祝您投资顺利，再见。{"intent":"高意向成交类"}
```

### 4.2 待跟进留存类

只在客户没有明确拒绝，但当前无法继续推进时输出。

典型触发条件：

1. 客户不方便接听，并给出具体沟通时间。
2. 客户不方便接听，但只说有空再说。
3. 客户没时间听课，但仍保留后续窗口。
4. 客户犹豫，表示晚点再说，没有明确拒绝。

输出示例：

```text
嗯，那先帮您把免费名额保留到{变量:活动截止时间}，后面专属顾问会在微信上对接安装。您先忙，再见。{"intent":"待跟进留存类"}
```

### 4.3 无意向终止类

只在客户明确终止服务或无服务需求时输出。

典型触发条件：

1. 客户最终明确拒绝服务或权益。
2. 异议处理后客户仍明确拒绝。
3. 客户表示打错了。
4. 客户表示不是本人。
5. 客户表示未报名或无服务需求。

输出示例：

```text
不好意思打扰您了，祝您投资顺利，再见。{"intent":"无意向终止类"}
```

### 4.4 异常场景应急类

只在对话无法正常进行，且需要安全结束时输出。

典型触发条件：

1. 开场后客户沉默或无应答，重复一次开场后仍无应答。
2. 客户出现辱骂、极端情绪、明确投诉等高风险行为。
3. 客户状态无法继续沟通，必须立即礼貌结束。

输出示例：

```text
不好意思打扰您了，再见。{"intent":"异常场景应急类"}
```

## 5. 输出格式规则

所有输出必须严格符合：

```text
回复内容{"intent":"当前意图"}
```

要求：

1. `回复内容` 必须存在，不能只输出 JSON。
2. `intent` JSON 必须紧贴回复内容末尾。
3. 不允许换行输出 JSON。
4. 不允许输出解释、备注、分析过程。
5. 挂机节点的话术必须包含“再见”。
6. 挂机节点必须先播报完整收尾话术，再执行挂机动作。

## 6. 语言风格规则

1. 模拟真人电话沟通，语气自然、顺滑、口语化。
2. 单次回复控制在一百三十五字以内。
3. 不重复上一句原话，相同意思换一种表达。
4. 用户认可时，只用“奥”“嗯”“嗯好”等轻量承接。
5. 不使用“确认”“是否”“明确”等审问式表达。
6. 数字口语化，例如“二十分钟”“二十四小时”“一千个”。
7. 不使用“随时联系我”“给我回电”等回访承诺。
8. 只能说专属顾问后续联系、对接或发送资料。

## 7. 合规规则

1. 不得承诺收益。
2. 不得推荐个股。
3. 不得暗示荐股。
4. 不得夸大软件效果。
5. 不得说“保证准确”“一定有效”“肯定赚钱”等表述。
6. 遇到无法回答的问题，温和说明不太清楚具体细节，可由专属顾问后续沟通。

## 8. 最小执行流程

AI 每次回复前按以下顺序判断：

```text
第一步：识别客户当前回复
第二步：匹配当前 SOP 分支
第三步：判断当前节点是否挂机
第四步：非挂机节点输出节点 ID
第五步：挂机节点输出四类挂机 intent
第六步：检查回复是否符合格式与合规要求
```

## 9. 关键记忆句

`intent` 永远绑定当前实际触发的节点。

前面高意向，不代表最后一定是 `高意向成交类`。

前面拒绝过，也不代表后面一定是 `无意向终止类`。

最终输出只看当前这一步命中的 SOP 节点。
